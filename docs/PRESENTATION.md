# Presentation outline (pitch deck)

Use this as a starting structure for meetup or hackathon slides. Adjust slide count to 6–8 as in the [build plan](build-plan.md).

## Suggested flow

1. **Title** — Zagreb Buddy + one-liner  
2. **Problem** — tourists vs locals (split or two bullets)  
3. **Solution** — chat agent + map + skills (one diagram: simplified architecture)  
4. **Demo** — screenshot or live: 2-hour street art + coffee scenario  
5. **How it works** — Next.js → VPS → Gemini; skills as JSON + markdown  
6. **Why us / why now** — PWA, local data, personality vs generic search  
7. **Roadmap** — see [future-vision.md](future-vision.md) (MVP → next → vision)  
8. **Thank you / repo / Q&A** — [github.com/squattedaccount/Zagreb-Buddy](https://github.com/squattedaccount/Zagreb-Buddy)

## Demo scripts

Practice the three scenarios in [demo-scenarios.md](demo-scenarios.md):

Target **3–5 minutes** spoken; keep one buffer slide or minute for live demo risk.
